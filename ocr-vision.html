Notes & caveats

Why per-engine Run buttons: loading all engines at once on mobile crashes the tab. Each engine loads on first Run; subsequent runs reuse the loaded model.

Detection: TF.js COCO-SSD (lite_mobilenet_v2). The COCO book class generally fires on spine-out books. If detection misses, the full frame is used.

Rotation: if the crop is taller than wide (typical for spine-out text), it's auto-rotated 90° counter-clockwise before OCR. If results look reversed, re-capture with the book flipped.

Tesseract Fast vs Best: same library, different trained data files. Fast (~11MB) prioritizes speed; Best (~25MB) prioritizes accuracy. Both run fully offline once cached.

PaddleOCR via @gutenye/ocr-browser wraps PP-OCR ONNX models (~10MB total) with onnxruntime-web. Often strong on stylized type and short text. Models load from the package's CD[...]

TrOCR models are transformer-based and produce the best results on stylized printed text or handwriting, but are ~120MB even quantized (q8). Choose printed for typeset spines, handwri[...]

All client-side. Nothing leaves the device. First run downloads weights (browser cache makes subsequent loads near-instant).
